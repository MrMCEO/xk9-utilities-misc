# BFGKazinoApp — правила разработки

## Git workflow

1. **Каждое изменение = коммит** с подробным описанием на русском:
   - что именно изменено
   - зачем / какую проблему решает
   - пример: `feat(minesweeper): изменить поле с 5x5 на 6x6, убрать безопасный первый клик`

2. **Масштабные изменения — в отдельной ветке:**
   - Новая игра → `feature/game-<name>`
   - Редизайн → `feature/redesign-<описание>`
   - Рефакторинг → `refactor/<описание>`
   - Ветка мержится в `main` только после проверки
   - Пример: `git checkout -b feature/new-casino-game`

3. **Мелкие правки** (текст, цвет, баг-фикс) — напрямую в `main`

## Агенты (команды)

- **По умолчанию** все агенты запускаются с `mode: "acceptEdits"` — авторедакт файлов без подтверждения пользователя
- Запрашивать подтверждение (`mode: "default"`) только если в промте явно указано обратное

### Состав команды по умолчанию

Если пользователь говорит "создай команду агентов" и не уточняет состав:

| Агент | Модель | Редактирует файлы | Роль |
|-------|--------|:-----------------:|------|
| **team-lead** | Opus 4.6 | ✅ | Координация, распределение задач |
| **frontend-dev** | Sonnet | ✅ | HTML/CSS/JS, app/v2/index.html |
| **backend-dev** | Sonnet | ✅ | Python бот, aiogram, SQLite |
| **reviewer** | Sonnet | ❌ | Проверяет код после исполнителей, даёт замечания |
| **tester** | Sonnet | ✅ (только тесты) | Пишет и запускает тесты, проверяет логику игр |
| **security** | Sonnet | ❌ | Анализ безопасности, ищет уязвимости |
| **docs** | Haiku | ✅ | Обновляет комментарии в коде и README |

### Порядок работы (pipeline)

1. **frontend-dev + backend-dev** — параллельно выполняют задачи
2. **reviewer + tester** — после завершения исполнителей, проверяют и сообщают замечания
3. **frontend-dev / backend-dev** — правят по замечаниям
4. **security** — финальная проверка безопасности
5. **docs** — добавляет комментарии и документацию по итогу

## Админ-панель

### Бот-команды для администраторов (ADMIN_IDS)

- **/admin** — открыть меню администратора (инлайн-клавиатура с опциями)
- **/promo CODE** — активировать промо-код (доступна всем пользователям)

### Опции в меню админ-панели (/admin)

| Кнопка | Функция | Что делает |
|--------|---------|-----------|
| 📊 Статистика | admin_stats | Показать ставки, доход казино, топ-игры за сегодня |
| 👥 Пользователи | admin_users | Постраничный список всех пользователей с балансами |
| 💰 Управление балансом | admin_balance | Изменить баланс пользователя по ID или @username (FSM-цепочка) |
| 📢 Рассылка | admin_broadcast | Отправить сообщение всем пользователям (FSM-цепочка) |
| 🎁 Промо-коды | admin_promos | Создавать/удалять коды, просмотреть статистику использования |
| 🔧 Режим обслуживания | admin_maintenance | Включить/выключить режим обслуживания (пользователи не могут играть) |
| 🏆 Топ игроков | admin_top | Вывести топ-5 по балансу и по количеству ставок |

### Web App админ-панель (скрытая, только для ADMIN_IDS)

Доступна через кнопку Web App, когда параметр `admin=1` передан в URL.

**Параметры:**
- `admin=1` — включить режим админа
- `admindata=<base64>` — JSON с данными (статистика, коды, ставки, график активности)

**Функции:**
- 📊 Агрегированная статистика (юзеры, ставки, доход, активные промо)
- 📈 График активности за 24 часа (ставки по часам)
- 🎁 Список промо-кодов (со статусом: активен/исчерпан)
- 🎮 История последних ставок (игрок, игра, ставка, результат)
- 🔧 Переключатель режима обслуживания (один клик)
- 💰 Изменение баланса пользователя (по ID)

### Создание промо-кода

1. Команда: `/admin` → 🎁 Промо-коды
2. Введите код (например: `BONUS100`)
3. Введите бонус в монетах (например: `1000`)
4. Введите макс. использований (например: `50`)

Пользователь активирует: `/promo BONUS100` и получит 1000 монет (один раз).

### Рассылка сообщения

1. Команда: `/admin` → 📢 Рассылка
2. Введите текст сообщения
3. Бот отправит всем пользователям (кроме забаненных)

### Банить / разбанить пользователя

1. `/admin` → 👥 Пользователи (выберите страницу)
2. Нажмите на иконку 🚫 (забанить) или ✅ (разбанить)
3. Забаненный пользователь не может играть или использовать промо-коды

### Режим обслуживания

**Включение через бот:**
- `/admin` → 🔧 Режим обслуживания → Включить

**Включение через Web App:**
- Админ-панель → кнопка 🔧 (один клик)

Во время обслуживания:
- ✅ Админы могут играть и проверять
- ❌ Обычные пользователи видят сообщение "Казино на обслуживании"
